# SnapClass - AI Attendance System

SnapClass is a Streamlit-based attendance system for classrooms. It gives teachers a dashboard to create subjects, share enrollment links, and mark attendance from classroom photos or voice recordings. Students can register with Face ID, optionally enroll a voice sample, join subjects with a code or QR link, and track their attendance.

## Features

### Teacher portal

- Register and log in as a teacher.
- Create and manage subjects with subject codes and sections.
- Share subjects using a join code, direct link, or QR code.
- Upload or capture classroom photos for AI-based face attendance.
- Review detected students before saving attendance.
- View attendance history grouped by class session.
- Optionally run voice-based attendance for students with saved voice embeddings.

### Student portal

- Log in using face recognition.
- Register a new profile from camera input.
- Store face embeddings and optional voice embeddings.
- Enroll in subjects using a teacher-provided code or join link.
- View enrolled subjects and attendance totals.
- Unenroll from subjects.

## Getting Started

### 1. Prerequisites

- Python 3.10 or 3.11 recommended.
- A Supabase project.
- A working webcam for student registration and photo capture.
- A working microphone for optional voice enrollment and voice attendance.

### 2. Create a virtual environment

```bash
python -m venv .venv
```

On Windows:

```bash
.venv\Scripts\activate
```

On macOS or Linux:

```bash
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

If `dlib-bin` fails on your platform, install the build tools required by `dlib` or replace it with a platform-compatible `dlib` installation.

### 4. Configure Streamlit secrets

Create `.streamlit/secrets.toml`:

```toml
SUPABASE_URL = "https://your-project.supabase.co"
SUPABASE_KEY = "your-supabase-anon-or-service-key"
```

Do not commit real Supabase secrets to a public repository.

### 5. Create Supabase tables

The app expects these table names:

- `teachers`
- `students`
- `subjects`
- `subject_students`
- `attendance_logs`

You can start with this SQL in the Supabase SQL editor:

```sql
create table if not exists teachers (
  teacher_id bigint generated by default as identity primary key,
  username text not null unique,
  password text not null,
  name text not null,
  created_at timestamptz not null default now()
);

create table if not exists students (
  student_id bigint generated by default as identity primary key,
  name text not null,
  face_embeddings jsonb,
  voice_embeddings jsonb,
  created_at timestamptz not null default now()
);

create table if not exists subjects (
  subject_id bigint generated by default as identity primary key,
  subject_code text not null unique,
  name text not null,
  section text not null,
  teacher_id bigint not null references teachers(teacher_id) on delete cascade,
  created_at timestamptz not null default now()
);

create table if not exists subject_students (
  id bigint generated by default as identity primary key,
  subject_id bigint not null references subjects(subject_id) on delete cascade,
  student_id bigint not null references students(student_id) on delete cascade,
  created_at timestamptz not null default now(),
  unique(subject_id, student_id)
);

create table if not exists attendance_logs (
  attendance_id bigint generated by default as identity primary key,
  student_id bigint not null references students(student_id) on delete cascade,
  subject_id bigint not null references subjects(subject_id) on delete cascade,
  timestamps timestamptz not null default now(),
  is_present boolean not null default false
);
```

For quick local development, make sure your Supabase Row Level Security policies allow the app to read and write these tables. For production, replace open policies with role-aware policies.

### 6. Run the app

```bash
streamlit run app.py
```

Open the local URL shown by Streamlit, usually:

```text
http://localhost:8501
```

## How It Works

### Teacher workflow

1. Open the Teacher Portal.
2. Register or log in.
3. Create a subject with a code, name, and section.
4. Share the subject code, join link, or QR code with students.
5. Add classroom photos from upload or camera capture.
6. Run analysis to detect registered students.
7. Review the attendance table.
8. Confirm and save attendance to Supabase.

### Student workflow

1. Open the Student Portal.
2. Capture a face photo with the camera.
3. If the face is not recognized, create a new profile.
4. Optionally record a voice sample during registration.
5. Enroll in a subject using a code or teacher share link.
6. View enrolled subjects and attendance stats.

## Deployment

This app can be deployed on Streamlit Community Cloud or another Python-friendly hosting provider.

For Streamlit Community Cloud:

1. Push the project to GitHub.
2. Create a new Streamlit app.
3. Set `app.py` as the entry file.
4. Add `SUPABASE_URL` and `SUPABASE_KEY` in the app secrets.
5. Deploy.

If you deploy to a different URL, update the hardcoded domain in:

```text
src/components/dialog_share_subject.py
```

The current share link domain is:

```text
snapclass-ai-attendance.streamlit.app
```

## Important Notes

- Face embeddings and voice embeddings are stored in Supabase as JSON arrays.
- Uploaded attendance photos are kept in Streamlit session state while attendance is being processed.
- The face classifier is cached with `st.cache_resource` and retrained after a new student profile is created.
- Camera and microphone features require browser permissions.
- Voice attendance only works for students who registered a voice embedding.

## Troubleshooting

### Supabase secret errors

Check that `.streamlit/secrets.toml` contains both keys:

```toml
SUPABASE_URL = "..."
SUPABASE_KEY = "..."
```

### dlib installation errors

Use Python 3.10 or 3.11 and a 64-bit environment. On some systems, you may need CMake and C++ build tools before installing `dlib`.

### Camera or microphone does not appear

Allow camera and microphone permissions in the browser. On deployed apps, use HTTPS.

### No face recognized

Make sure the student has registered a clear face image, the classroom photo is well lit, and the face is visible from the front.

### Attendance is not saving

Confirm that the `attendance_logs` table exists, the `timestamps` column is present, and the Supabase key has permission to insert rows.
